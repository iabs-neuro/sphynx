# CreatePresetApp — руководство пользователя (рус.)

Пошаговая инструкция по созданию пространственной разметки одной
сессии видео.

## Что нужно

- Видеофайл сессии (`.mp4`, `.avi`, `.mov`)
- Знание одного-двух реальных расстояний на арене (длина стенки в см)
- Папка, куда сохранять результаты

## 1. Запуск приложения

```matlab
startup
sphynx.app.CreatePresetApp()
```

Окно открывается развёрнутым на весь экран. Две вкладки: **Create
Preset** (про неё ниже) и **Analyze Session** (заглушка для будущего
batch-run UI).

## 2. Блок 1 — Load

В каждой колонке: сверху кнопка **Browse**, снизу короткое поле
с текущим путём.

- **Project root** — стартовый каталог для остальных диалогов.
- **Video** — твоё видео. В Preview появится первый кадр.
- **Output dir** — куда сохранять. Приложение автоматически
  создаст подпапку с именем видео.
- **Preset** (опционально) — подгрузка существующего пресета,
  чтобы переиспользовать калибровку.

## 3. Блок 2 — Calibration

Перевод пикселей в см по 4 опорным точкам.

1. В **cm Y** введи реальное расстояние в см между точкой 1 и точкой 2
   (вертикальная пара).
2. В **cm X** — между точкой 3 и точкой 4 (горизонтальная пара).
3. Нажми **Choose points** (красная). Откроется окно с кадром.
4. Кликни 4 точки в порядке:
   - точка 1 — верх вертикальной пары
   - точка 2 — низ вертикальной пары
   - точка 3 — лево горизонтальной пары
   - точка 4 — право горизонтальной пары

   Окно закроется автоматически.
5. Нажми **Compute** (красная). Появятся пиксели/см по Y и X отдельно
   (`Y:`, `X:`), среднее (`avg:`) и поправочный коэффициент `kcorr:`.
   Если `kcorr` сильно отличается от 1 — порядок кликов был неверный.

Выбери **Exp** (Experiment type) из выпадающего списка.

## 4. Блок 3 — Arena

1. Нажми одну из кнопок геометрии (жёлтые): **Polygon**, **Circle**,
   **Ellipse**, **O-maze**. Активная подсвечивается.
2. **Pick arena points** (красная).
3. В отдельном окне кликни точки границы, потом ENTER. Окно
   закроется.
   - Polygon — углы.
   - Circle — минимум 3 точки на ободе.
   - Ellipse — минимум 5 точек на ободе.
   - O-maze — 3+ на ВНЕШНЕМ ободе, ENTER, потом 3+ на ВНУТРЕННЕМ,
     ENTER.

Арена отрисуется чёрной линией в Preview.

## 5. Блок 4 — Objects

1. Выбери геометрию (Polygon / Circle / Ellipse).
2. **+ Add** (красная). Клики, ENTER.
3. Появится диалог "Is Object1 correct?" с тремя кнопками:
   - **Yes** — оставить.
   - **No (redo)** — убирает объект и переоткрывает picker. Старая
     маска **не остаётся** на Preview.
   - **No (delete)** — удаляет без переспроса.

   Диалог переспрашивает пока не **Yes** или **No (delete)**.
4. Чтобы убрать или переделать существующий объект, выдели его в
   списке, нажми **Remove**, **Replace** или **Rename**.
   - **Remove** — удалить.
   - **Replace** — переоткрыть picker для маски этого объекта
     (метка сохраняется).
   - **Rename** — переименовать.

Выбранный объект подсвечивается оранжевым на Preview.

## 6. Блок 5 — Zones

Можно применить несколько стратегий — они складываются.

1. Выбери **Strategy**:
   - `corners-walls-center` — квадратные/полигональные арены.
     Введи **Wall** (cm).
   - `strips` — N равных полос, горизонтальных или вертикальных.
     **Strips** (число) и **Dir**.
   - `circle-rings` — круглые арены. **Wall** и **Middle**.
   - `none` — без разбиения.
2. Если есть объекты, задай **Obj zone** (cm) — радиус инфляции
   зоны взаимодействия вокруг объекта.
3. **Preview** — увидишь предлагаемое разбиение в Preview (контурами
   разными цветами).
4. **Add to set** — фиксирует. Лейбл "Added: ..." обновится тегом
   стратегии (например, `corners-walls-center + strips_horizontal_3`).
5. **Clear** — снести все зоны.

Зоны **автоматически очищаются** (с предупреждением в Log) когда
ты двигаешь / вращаешь / переименовываешь / replace / добавляешь
объект — потому что зоны, построенные на старой геометрии, больше
не совпадают с новой.

## 7. Блок 6 — Save

- **Save preset** (красная) сохранит
  `<output_dir>/<videobase>/<videobase>_Preset.mat` И автоматически
  сохранит общую картинку разметки рядом
  (`<videobase>_layout.png`).
- Если стоит галочка **plot all zones** — также сохранит PNG
  для каждой зоны отдельно (`<videobase>_zone_<name>.png`).

## 8. Использовать пресет на другом видео

Если камера примерно та же между днями:

1. **Browse** на строке **Video** — выбери новое видео. Калибровка,
   арена, объекты, зоны остаются.
2. Если всё совпадает — сразу **Save preset** (подпапка будет с
   новым именем).
3. Если что-то сместилось — пользуйся полосой под Preview:
   - **Target**: `All`, `Arena`, `Object1`, `Object2`, ...
   - **Step (px)** — шаг в пикселях (или градусах для поворота).
   - **Left / Right / Up / Down** — сдвинуть.
   - **Rot ↺ / Rot ↻** — повернуть вокруг центра выбранной маски.
     Примечание: с target=`All` сейчас каждая маска вращается вокруг
     своего центра (см. `TODO.md` #6 — будет починено).
4. После подгонки — **Preview** или **Add to set** в Zones (маски
   автоматически пересчитываются из обновлённых контуров).
5. **Save preset**.

## 9. Где помощь внутри приложения

На каждой панели есть кнопка **INFO** (голубоватая). Открывает
короткое окошко с инструкцией к этой панели.

## 10. Log

Справа внизу — прокручиваемый текстовый блок, дублирующий вывод
MATLAB Command Window для приложения. Последние 500 строк хранятся.
*Автопрокрутка вниз работает с R2021a+; в R2020a нужно прокручивать
вручную (см. `TODO.md` #2).*

## 11. Если что-то не работает

- **Preview пустой** — сначала загрузи видео.
- **"Need at least 3 corners"** — кликни больше точек перед ENTER.
- **kcorr далеко от 1** — порядок кликов был неверный (сначала
  Y-пара, потом X-пара).
- **Маски Object Out выглядят как сплошные диски** — должны быть
  кольцами. Убедись, что **Obj zone (cm)** > 0.
- **Зоны исчезли после move/rotate** — это by design: автоматически
  очищаются, потому что старые маски зон больше не совпадают с новой
  геометрией. Re-Add to set когда закончишь подгонять.
