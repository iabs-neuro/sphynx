# CreatePresetApp — руководство пользователя (рус.)

Пошаговая инструкция по созданию пространственной разметки для одной
сессии видео.

## Что нужно

- Видеофайл сессии (`.mp4`, `.avi`, `.mov`)
- Знание одного-двух реальных расстояний на арене (длина стенки в см)
- Папка, куда сохранять результаты

## 1. Запуск приложения

```matlab
startup
sphynx.app.CreatePresetApp()
```

Откроется окно с двумя вкладками: **Create Preset** и **Analyze
Session**. Вкладка Analyze Session — заглушка на будущее. Всё
ниже — на первой вкладке.

## 2. Блок 1 — Load

- Нажми **Browse** у поля **Project root** и выбери папку, которой
  ты пользуешься постоянно. С неё стартуют все остальные диалоги.
- Нажми **Browse** у поля **Video file**, выбери видео.
  В Preview появится первый кадр.
- Нажми **Browse** у поля **Output dir**, выбери куда складывать
  пресеты и картинки. Приложение автоматически создаст подпапку
  с именем видео.
- Опционально: **Existing preset** если хочешь подгрузить
  калибровку из предыдущего пресета.

Когда в Preview видно кадр — можно калиброваться.

## 3. Блок 2 — Calibration

Перевод пикселей в сантиметры по 4 опорным точкам.

1. В поле **cm Y** введи реальное расстояние в см между точкой 1 и
   точкой 2 (вертикальная пара).
2. В поле **cm X** введи реальное расстояние в см между точкой 3 и
   точкой 4 (горизонтальная пара).
3. Нажми **Calibration. Choose points**. Откроется окно с кадром.
4. Кликни 4 точки в таком порядке:
   - точка 1 — верх вертикальной пары
   - точка 2 — низ вертикальной пары
   - точка 3 — лево горизонтальной пары
   - точка 4 — право горизонтальной пары

   Окно закроется автоматически после 4-го клика.
5. Нажми **Calibration. Calculation**. Покажутся пиксели/см по Y и
   X отдельно (`Y:` и `X:`), среднее (`avg:`) и поправочный
   коэффициент `kcorr:`. Если `kcorr` сильно отличается от 1.000
   (больше пары процентов) — возможно ты кликнул точки в неправильном
   порядке.

Выбери **Experiment type** из выпадающего списка.

Если нужно посмотреть другой кадр — кнопка **Next frame** под Preview.

## 4. Блок 3 — Arena

1. Нажми одну из кнопок геометрии: **Polygon**, **Circle**,
   **Ellipse**, **O-maze**. Активная подсветится жёлтым.
2. Нажми **Pick arena points** (красная).
3. В отдельном окне кликни точки границы арены, потом ENTER:
   - **Polygon**: углы. Приложение замкнёт полигон.
   - **Circle**: минимум 3 точки на ободе.
   - **Ellipse**: минимум 5 точек на ободе.
   - **O-maze**: 3+ точки на ВНЕШНЕМ ободе, ENTER, потом 3+ точки
     на ВНУТРЕННЕМ ободе, ENTER.

Граница арены отрисуется чёрной линией в Preview.

## 5. Блок 4 — Objects

Если в сессии есть объекты (миски, новые предметы и т.д.):

1. Выбери геометрию (Polygon / Circle / Ellipse).
2. Нажми **+ Add** (красная).
3. Кликни точки границы объекта, ENTER.
4. Появится диалог: "Is Object1 correct?" с тремя кнопками:
   - **Yes**: оставить.
   - **No (redo)**: убирает объект, переоткрывает picker. Старая
     маска **не остаётся** на Preview во время повтора.
   - **No (delete)**: удаляет без переспроса.

Диалог переспрашивает пока не **Yes** или **No (delete)**.

Чтобы удалить или переделать существующий объект — кликни его в
списке, потом **Remove** или **Replace**. Выбранный объект
подсвечивается оранжевым на Preview.

## 6. Блок 5 — Zones

Можно применить несколько стратегий — они складываются.

1. Выбери стратегию:
   - `corners-walls-center` — для квадратных/полигональных арен.
     Поставь **Wall (cm)** — ширина пристеночной зоны.
   - `strips` — деление арены на N равных полос. Введи **N strips**
     и **Strip dir**.
   - `circle-rings` — для круглых арен. **Wall (cm)** и **Middle
     (cm)**; приложение построит wall, опционально middle1/2/...
     и центральный диск.
   - `none` — без разбиения.
2. Если есть объекты — задай **Object zone (cm)**: радиус инфляции
   зоны взаимодействия вокруг объекта.
3. Нажми **Preview zones** (красная) — увидишь предлагаемое
   разбиение на Preview в magenta.
4. Если устраивает — **Add to set** (красная). Зоны фиксируются.
   Счётчик "Committed zones" обновится.
5. Если хочешь сбросить — **Clear all** (красная).

Можно вызывать **Add to set** несколько раз с разными стратегиями —
они комбинируются.

## 7. Блок 6 — Save / Plot

- **Save preset** (красная): запишет `.mat` в
  `<output_dir>/<videobase>/<videobase>_Preset.mat` и автоматически
  сохранит общую картинку разметки рядом.
- **Make plot** (красная): сохранит общую картинку. Если стоит
  галочка **plot all zones** — добавит ещё PNG для каждой зоны
  отдельно.

## 8. Использовать пресет на другом видео

Если камера примерно та же между днями:

1. Нажми **Browse** для **Video file** и выбери новое видео.
   В Preview появится новый кадр. Калибровка/арена/объекты/зоны
   остаются прежними.
2. Если всё совпадает с новым видео — сразу **Save preset**
   (подпапка будет с новым именем автоматически).
3. Если что-то сместилось — пользуйся **Move / Rotate** под Preview:
   - **Target**: выбор Арены или конкретного объекта
   - **Step (px)**: шаг в пикселях (или градусах для поворота)
   - **Left / Right / Up / Down**: сдвинуть
   - **Rot CCW / Rot CW**: повернуть вокруг центра масок
4. После подгонки — **Refit mask** чтобы пересчитать бинарную маску
   из обновлённого контура.
5. **Save preset**.

## 9. Где помощь внутри приложения

Кнопка `INFO` (бледно-голубая) есть на каждой панели. Открывает
короткое окошко с инструкцией.

## 10. Если что-то не работает

- **Preview пустой**: сначала загрузи видео. Без видео не покажется.
- **"Need at least 3 corners"**: для полигона нужно >=3 точки.
- **kcorr далеко от 1**: возможно ты кликнул точки калибровки в
  неправильном порядке (сначала Y-пара, потом X-пара).
- **Маски объектов выглядят не так на сохранённой картинке**:
  кольцевая зона `Object<N>Out` должна выглядеть как кольцо (с
  дыркой), а не сплошной диск. Если выглядит сплошной — проверь,
  что `ObjectZoneWidthCm` больше нуля.
- **Сообщения статуса**: появляются в Command Window MATLAB с
  префиксом `[INFO] [App] ...`.
